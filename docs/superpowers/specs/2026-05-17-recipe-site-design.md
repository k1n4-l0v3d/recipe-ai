# Сайт с рецептами + AI ассистент — Дизайн-спецификация

**Дата:** 2026-05-17  
**Статус:** Согласовано

---

## Обзор

Кулинарный сайт с встроенным AI-ассистентом на базе Groq. Пользователь просматривает каталог рецептов по категориям и параллельно общается с AI в чат-панели. AI генерирует ответы из своих знаний и при необходимости ищет актуальные рецепты в интернете через Tavily.

---

## Архитектура

### Монолит: Go + React SPA

Go бэкенд выполняет две функции:
1. Отдаёт собранный React билд как статику (`/`)
2. Предоставляет REST API (`/api/*`)

Один бинарник → один сервис на Render. Нет CORS-проблем, один деплой.

```
Браузер
  └── React SPA (Framer Motion, анимации)
        ↕ REST/SSE
Go Backend (Gin)
  ├── Static: отдаёт React build
  ├── GET  /api/categories        — список категорий
  ├── GET  /api/categories/:id    — рецепты категории (Groq генерирует список)
  ├── GET  /api/recipes/:id       — конкретный рецепт (Groq генерирует)
  └── POST /api/chat (SSE)        — AI чат со стримингом
        ├── Groq API (llama-3.3-70b-versatile, tool use, streaming)
        └── Tavily API (вызывается как tool при необходимости)
```

### Переменные окружения

```
GROQ_API_KEY=...
TAVILY_API_KEY=...
PORT=8080
```

## Функциональность

### Главная страница

- Hero-секция с поисковой строкой (поиск через AI)
- Сетка категорий: Супы, Паста, Салаты, Мясо, Рыба, Десерты, Выпечка, Завтраки
- При клике на категорию → Groq генерирует список из 6-8 рецептов
- Анимированное появление карточек (Framer Motion stagger)

### Страница рецепта

Разделена на два столбца:

**Левая панель (рецепт):**
- Название, кухня, время приготовления
- Теги (острое, веган, быстро и т.д.)
- Список ингредиентов с количеством
- Пошаговое приготовление
- Источники (ссылки на форумы если Tavily нашёл)

**Правая панель (AI чат):**
- История диалога с контекстом текущего рецепта
- Стриминговый ввод ответа (токен за токеном)
- Поле ввода внизу
- AI знает какой рецепт открыт — отвечает в контексте

### AI-поведение (Groq)

Системный промпт определяет роль кулинарного эксперта. Groq получает инструмент `search_recipes(query: string)`:

- **Без поиска:** вопросы о замене ингредиентов, техниках, советы — Groq отвечает сразу
- **С поиском:** запросы на новые рецепты, актуальные тренды — Groq вызывает Tavily, Go выполняет запрос, результаты возвращаются Groq, AI формирует финальный ответ со ссылками

## Визуальный стиль: Тёмный гурман

- **Фон:** `#0d0d0d` (почти чёрный)
- **Акцент:** `#ff6b35` (огненно-оранжевый)
- **Вторичный:** `#e0a060` (золото)
- **Текст:** `#ffffff`, `#888888` (приглушённый)
- **Градиенты:** радиальные свечения оранжевого на тёмном фоне

### Анимации (Framer Motion)

- Карточки рецептов: stagger-появление при загрузке категории
- Чат-сообщения: slide-in снизу
- Hero: floating-эффект на фоновых элементах
- Переходы между страницами: fade + slide

## Деплой на Render

`render.yaml`:
```yaml
services:
  - type: web
    name: recipe-ai
    runtime: go
    buildCommand: "cd web && npm install && npm run build && cd .. && go build -o server ./cmd/server"
    startCommand: "./server"
    envVars:
      - key: GROQ_API_KEY
        sync: false
      - key: TAVILY_API_KEY
        sync: false
```

Go при старте ищет `./web/dist` и отдаёт как статику.

## Ограничения и решения

| Проблема | Решение |
|---|---|
| Groq rate limits | Показывать индикатор загрузки, graceful error |
| Tavily платный | Free tier 1000 req/мес, достаточно для MVP |
| Холодный старт Render | Render держит сервис живым на платном плане; на free — допустимо |
| Большой контекст чата | Хранить последние 10 сообщений в истории |
