# Combo Selector — Выбор основного блюда и гарнира

## Цель

Добавить на главную страницу секцию ниже категорий, где пользователь выбирает основное блюдо и/или гарнир, нажимает «Найти рецепты» и получает список сгенерированных рецептов под этот состав.

## UI — компонент ComboSelector

**Файл:** `web/src/components/ComboSelector.tsx`

**Расположение:** В `Home.tsx` сразу после `<CategoryGrid>`, перед блоком с рецептами.

**Структура:**
- Заголовок секции: «Подбери по составу» (стиль как у «Или выбери сам»)
- Строка «Основное блюдо» — чипсы: Курица, Говядина, Свинина, Рыба, Морепродукты, Баранина, Индейка, Грибы, Тофу + «✏️ Другое»
- Строка «Гарнир» — чипсы: Рис, Картофель, Паста, Гречка, Овощи, Булгур, Чечевица, Пюре, Салат + «✏️ Другое»
- При клике «✏️ Другое» — чипсы строки прячутся, появляется текстовое поле; при очистке поля — чипсы возвращаются
- Выбранный чип подсвечивается оранжевым (как активная категория)
- Кнопка «Найти рецепты →» — disabled если оба поля пусты
- Раскладка вертикальная: основное → гарнир → кнопка (справа)

**Взаимодействие с категориями:**
- Выбор категории сбрасывает комбо-результаты и состояние ComboSelector
- Нажатие «Найти рецепты» сбрасывает выбранную категорию и её рецепты
- Два режима взаимоисключают друг друга

**Загрузка:**
- Пока грузится — показываем те же `SkeletonCard` что и у категорий
- После — сетка `RecipeCard` с кнопкой «Ещё рецепты»
- «Ещё рецепты» передаёт `exclude` из уже показанных названий

## API — новый клиентский метод

**Файл:** `web/src/api/client.ts`

Добавить метод:
```ts
getComboRecipes(main: string, side: string, exclude?: string[]): Promise<RecipeSummary[]>
```

Вызывает `GET /api/recipes/combo?main=Курица&side=Картофель&exclude=...`

## Бэкенд — новый эндпоинт

**Маршрут:** `GET /api/recipes/combo`

**Файл:** `internal/handlers/categories.go` — новая функция `GetComboRecipes`

**Параметры:**
- `?main=` — название основного блюда (опционально)
- `?side=` — название гарнира (опционально)
- `?exclude=` — запятая-разделённый список уже показанных блюд (опционально)
- Если оба `main` и `side` пусты — возвращает 400

**Логика построения категории для Groq:**
- Оба заполнены: `"блюда из {main} с {side}"`
- Только main: `"блюда из {main}"`
- Только side: `"блюда с {side}"`

**Вызов:** `gen.GenerateRecipeListExcluding(ctx, categoryStr, exclude)` — без изменений в Groq-клиенте.

**Ответ:** тот же формат `[]RecipeSummary` что и `/api/categories/:id/recipes`

**Регистрация маршрута:** `cmd/server/main.go` рядом с остальными API-роутами.
